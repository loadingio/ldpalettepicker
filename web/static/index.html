ldPalettePicker

palette utilities including editor, picker and palette wrapper.

About

ldPlaettePicker is a set of tool for manipulating color palettes. It contains three parts:

  • ldPalette ( ldpalette ) - Palette wrapper, for manipulating palette.
  • ldPaletteEditor ( ldpe ) - API and interface for editing palette.
  • ldPalettePicker ( ldpp ) - Widget for picking and editing palette.

Usage

ldPalette is designed to be used along with Pug, so in the following examples we will use Pug instead of HTML. To use ldPalettePicker, download and include the required file:

include path-to/ldpp.pug link(rel="stylesheet",type="text/css",href="path-to/ldpp.css") script(src="path-to/ldpp.js")

ldPalette

There are two parts of ldPalette: ldPalette JS Object, and ldPalette HTML Construct.

a ldPalette object is like this:

{ name: "Palette Name", tag: ["tag", "list", ...], # either one of below colors: [ldcolor, ldcolor, ...] # type agnoistic color. we should internally use this colors: [{hex: "#999", tag: [...]}, ...] # hex. color follows ldcolor format. colors: ["#999", ...] # compact but losing color tags # deprecated and we should use ldcolor directly instead of a indirect object with value member. colors: [{value: ldcolor,...}, {value: ldcolor, ...}] }

For visualizing a palette in HTML, use ldp class with following structure:

.ldp - container of this palette. .colors - wrapper for colors div .ctrl - contains buttons shown when hovering. div(data-action="...") - button for triggering certain action .color - blocks colored by setting background attribute in inline style. .name - optional name containing the name of this palette.

Following is an example of a palette:

.ldp .colors .ctrl div(data-action="view") #[i.i-eye]View div(data-action="edit") #[i.i-chevron-down]Export .colors .color(style="background:#ff0") .color(style="background:#ff0") ... .name Palette

Customization

ldp provides the basic styles and DOM construct. To further styling it, you can customize it yourself by CSS. ldPalette provides some CSS classes for basic variations, It's at your discretion to use them.

Here are some common CSS styles you'd probably like to customize:

  • .ldp - width for width of palette.
  • .ldp - font-size for font size of palette.
  • .colors - height for height of palette.

The size, by default, is proportional to container font-size and designed to match the size of bootstrap form-control, but it's always customizable. Control the size by applying one of the ldp-lg, ldp-sm CSS classes.

API

ldPalette ships along with some helper APIs:

  • ldpalette.convert(palette, format) - convert palette to certain format, and return corresponding object. 4 formats are currently available:
    • SVG
    • PNG
    • JSON
    • SCSS
  • ldpalette.download(palette, format) - trigger downloading of a converted palette.

ldPaletteEditor

ldPalettePicker ships with a bunch of pre-built palettes, but sometimes user just need to customize their own. This could be done through the ldPaletteEditor.

ldPaletteEditor is more complicated than simply a palette, so we prepare the DOM construct via a Pug mixin:

.ldpe: +ldpe script. new ldpe({ root: '.ldpe' });

ldPaletteEditor provides following APIs:

  • getPal - get the current palette, in ldPalette format.
  • init(opt) - initialize editor. opt is an object with following member:
    • pal - default palette, in ldPalette format.
  • undo - undo the last edit action.
  • clearLog - clear edit history.

ldPalettePicker

ldPalettePicker is an integrated widget for choosing, managing and editing palettes. It contains at least 3 tabs: built-in palettes, saved palettes, and edit panel.

To create a ldPalettePicker, it's similar to how to prepare a ldPaletteEditor; prepare the HTML with Pug mixin, and initialize it with JavaScript. However, there are two notes about initializing ldPalettePicker:

  • We will need to set container's height for ldPalettePicker to be properly shown, minimal 570px:
  • We will need to prepare some builtin palettes to be listed in the default panel. ldPalettePicker ships with a default list of palette so we will use it to demonstrate here.
.ldpp(style="height:570px") +ldpp script. new ldpp({ root: '.ldpp', palettes: ldpp.get( 'default' ) });
